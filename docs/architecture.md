# nxtLLM V0.1 架构设计文档

## 1. 系统总览

nxtLLM 是面向大语言模型（LLM）推理场景的高性能内存管理引擎。核心目标是解决 LLM 推理过程中 KV-cache 和模型权重对显存/内存的巨大压力，通过分层存储和智能驱逐策略实现高效的页面级内存管理。

### 1.1 设计目标

| 目标 | 说明 |
|------|------|
| 高效内存利用 | 三层缓冲池（GPU HBM → CPU DRAM → NVMe SSD）统一管理 |
| 智能驱逐 | LRU-K (K=3) 算法避免热点页面被误驱逐 |
| 请求感知 | 每请求页面目录 + 准入控制，防止内存超载 |
| 后台碎片整理 | 异步合并空闲页面，减少内存碎片 |
| 模块化分层 | 5 层架构，各层独立可替换 |

### 1.2 五层架构

```
┌─────────────────────────────────────────────────────┐
│  Layer 5: 推理引擎                                    │
│  inference_runner.c / kv_cache.c / sampler.c          │
│  [V0.1 未实现]                                        │
├─────────────────────────────────────────────────────┤
│  Layer 4: 请求调度器                                   │
│  scheduler.c / admission.c                            │
│  [V0.1 未实现]                                        │
├─────────────────────────────────────────────────────┤
│  Layer 3: 页面分配器          ← V0.1 已实现           │
│  memory_manager.c                                     │
│  页面分配/释放、空闲链表、碎片整理                       │
├─────────────────────────────────────────────────────┤
│  Layer 2: 驱逐管理器          ← V0.1 已实现           │
│  lru_k.h / lru_k.c                                    │
│  LRU-K (K=3) 驱逐、时间戳追踪、victim 选择             │
├─────────────────────────────────────────────────────┤
│  Layer 1: 分层缓冲池          ← V0.1 已实现           │
│  page_pool.h                                          │
│  GPU HBM / CPU DRAM / NVMe SSD 三层存储               │
└─────────────────────────────────────────────────────┘
```

## 2. 模块依赖关系

```
page_pool.h ─────► lru_k.h
     │                  │
     ▼                  ▼
memory_manager.c    lru_k.c
     │                  │
     └────┬─────────────┘
          ▼
    test_page_pool.c
```

- `page_pool.h` 定义所有核心数据结构和 API
- `lru_k.h` 定义 LRU-K 算法接口，被 `page_pool.h` 引用
- `memory_manager.c` 实现页面分配、驱逐、准入控制
- `lru_k.c` 实现 LRU-K 时间戳管理
- `test_page_pool.c` 覆盖所有模块的单元测试

## 3. 构建系统

- **语言标准**: C11
- **构建工具**: CMake >= 3.16
- **编译选项**: `-Wall -Wextra -g`
- **产物**:
  - `nxtllm_core` — 静态库（memory_manager.c + lru_k.c）
  - `test_page_pool` — 测试可执行文件
